The content of the simple theme look like this

📁 css css files go here
📁 images theme images goes here
📁 templates twig files for overrides
📄 mytheme.info.yml it is only requred file for theme, specifyng:
  • theme name
  • type of project (module or theme)
  • core compatibility
  • regions and feature overrides
  • location of css and other nedeed files
📄 mytheme.libraries.yml add css and js files in containers, which than must be included in info.yml file, or decraring dependencies like core assets, as jquery.
📄 mytheme.breakpoints.yml activate Responsive Images in Drupal > Extends, then breakponts would appear in Drupal > configuration > Responsive Image Styles (drupal.org)
📄 mytheme.theme Used to store theme-specific PHP functions, preprocess functions and hooks
project names are important, they are the "key" that connectrs all components of the project, name must be unique, no spaces, dashes or punctuation marks, just simple lowercase word
